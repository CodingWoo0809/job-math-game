## 1. 게임 개요

WH가 집에서 학교까지 이동하며 18개의 직무 상황 기반 난관을 수학적으로 해결하는 2D 싱글 플레이 웹 어드벤처 게임이다. 플레이어는 문제의 답만 입력하는 것이 아니라 환경을 탐색하고, 필요한 정보를 골라 연결하고, 계산 결과를 실제 행동에 적용한다.

게임은 직무수학의 4개 단원에 대응하는 4개 챕터로 구성한다. 성취기준 수에 맞추어 챕터 1에는 3개 맵, 챕터 2~4에는 각 5개 맵을 배치하며, 전체 여정이 진행될수록 정보의 양과 수학적 사고 단계가 증가한다.

## 3. 챕터 구성

| 챕터 | 임시 이름 | 관련 단원 | 맵 | 현재 상태 |
|---|---|---|---|---|
| 1 | 수와 연산 | 01. 수와 연산 | 1map1~1map3 | 3개 맵 구현·검증 중 |
| 2 | 변화와 관계 | 02. 변화와 관계 | 2map1~2map5 | 5개 맵 구현·검증 중 |
| 3 | 도형과 측정 | 03. 도형과 측정 | 3map1~3map5 | 챕터 전환·3map1~3map5 구현·검증 중 |
| 4 | 자료와 가능성 | 04. 자료와 가능성 | 4map1~4map5 | 챕터 전환·4map1~4map5 구현·검증 중 |

단원명은 문서상 임시 챕터명이다. 각 챕터의 장소와 중심 사건이 정해지면 게임 세계에 어울리는 이야기형 이름으로 변경한다.

## 4. 전체 이야기 흐름

1. WH가 아침에 집을 나서 학교로 향한다.
2. 등굣길 곳곳에서 직무를 수행하는 사람들이 예상하지 못한 문제를 겪는다.
3. WH는 현장의 정보와 도구를 활용해 수학적으로 문제를 해결한다.
4. 해결 결과로 막힌 길이 열리거나 새로운 이동 수단과 경로가 생긴다.
5. WH는 학교에 도착한 뒤 자신이 사용한 수학이 학교에서 배우는 내용과 연결되어 있음을 깨닫는다.
6. 수학이 실제 생활과 직무를 더 정확하고 효율적으로 만드는 도구라는 WH의 깨달음으로 이야기가 끝난다.

## 5. 공통 플레이 구조

`도착 → 사건 발견 → 정보 탐색 → 수학적 모델링 → 해결안 실행 → 결과 확인 → 다음 맵 이동`

모든 맵은 이 구조를 공유하되, 정보의 형태와 조작 방식, 해결 전략 및 성공 결과는 서로 달라야 한다.
